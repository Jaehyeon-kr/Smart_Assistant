## 📋 2. 개요

### 프로젝트 배경 및 목표

**LecQ(대학)'는 대학생의 PDF 기반 학습 플랫폼입니다.**

대학생들은 강의 슬라이드 PDF를 업로드하면, AI 튜터를 통해 학습을 지원받는 플랫폼을 개발했습니다. 기존의 PDF 리더는 단순 열람 기능에 불과한 반면, 능동적 학습에 필요한 요약, 문제 생성, AI 튜터 상호작용 기능이 전무합니다. 학습들은 변역기·LLM 문제 은 노트 엽 등 여러 도구를 방강이 사용해야 하며, 이로 인해 학습 메력이 단절되고 효율이 저하됩니다.

LecQ는 위 다섯 가지 문제를 담합 플랫폼에서 해결합니다. PDF를 한 번 업로드하면 요약·문제·노트 AI 튜터가 등일 되인에서 즉시 제공됩니다. 학습은 이유 어떤 외부 도구 없이 LecQ 안에서 학습 전 과정을 완결합니다.

### 핵심 시스템 요구사항

| 항목 | 내용 |
|------|------|
| **서비스 유형** | 웹 브라우저 기반 (데스크탑 1280px 이상) |
| **기술 스택** | React · Node.js/Express · SQLite(개발) / PostgreSQL(배포) · Claude API |
| **대상 사용자** | 강의자료 PDF를 학습하는 대학생 |
| **핵심 목표** | PDF 1페이 입로드 → 요약·문제·노트 자동 생성, AI 튜터 기능 제공 |
| **AI 연동** | Claude API (claude-sonnet-4-20250514) — 요약·문제 생성·AI 튜터 |
| **화면 구조** | 좌우 2단: PDF 뷰어(60%) + 우측 탭 패널(40%): 요약/문제/노트/AI튜터 탐업 |

## 📌 3. 문제 정의

### 3-1. 문제 상황

대학생들은 강의 슬라이드 PDF를 많이 다루는데, 다음과 같은 어려움이 있습니다:

| # | 핵심 문제 | 구체적 상황 |
|---|---------|-----------|
| ① | **언어 장벽** | 영어 PDF를 읽을 때마다 변역 탭을 열어야 한다. 문장 단위 변역 과 원문 비교가 불가능하여 내용 파악이 어렵다. |
| ② | **수동적 학습** | 핵심 내용 정리 없이 텍스트를 수동으로 읽는 방식에 머무른다. 스스로 요약하거나 자기 정점하는 능동적 학습 행동이 부립되지 않는다. |
| ③ | **시점 준비 비용이** | PDF에서 직접 언급 문제를 만든 수 없이 개별 온라인 플랫폼의 의존해야 한다. 같은 내용과 주관한 문제로 학습 효과가 떨어진다. |
| ④ | **즉시 질문 불가** | PDF를 읽다가 이해되지 않는 개념이 생기도 즉시 질문할 수 있는 수단이 없다. ChatGPT 등 외부 도구 켜기(또는 페이지의 내용인 지)이 소실된다. |
| ⑤ | **도구 파편화** | 변역기·LLM 문제 은 노트 엽을 분도 간 벽금 공유가 없어 중용이 입혹해야 한다. |

### 3-2. 해결 방안

LecQ는 위 다섯 가지 문제를 담합 플랫폼에서 해결합니다. PDF를 한 번 입로드하면 요약·문제 노트·AI 튜터가 등일 되인에서 즉시 제공됩니다. 학습은 이유 어떤 외부 도구 없이 LecQ 안에서 학습 전 과정을 완결합니다.

#### 한국어 요약 자동 생성 (FR-10, 문제 ①②)
- PDF 입로드 즉시 Claude API가 전체 내용을 한국어 요약으로 변환합니다.
- 영어 PDF는 변역+요약을 단일 API 호출로 처리합니다(언어 감지 → 프롬프트 분기).
- 사용자는 요약 탭에서 즉시 핵심 내용을 파악할 수 있다.

#### 객관식 문제 자동 생성 (FR-11, 문제 ③)
- PDF 입로드 직후 Claude API가 4선지 객관식 문제 5개를 생성합니다. 반니 문제 구조는 PDF 콘텐츠와 일치하여 학습 효과가 높습니다.

#### AI 튜터 챗봇 (FR-13, 문제 ④)
- 현재 열린 PDF의 extracted_text를 기반으로 사용자 질문에 Claude API가 스트리밍 응답합니다.
- PDF 맥락이 자동으로 포함되어 질문 맥락이 명확합니다.

#### 단일 플랫폼 통합 (문제 ⑤)
- 최좌 PDF 뷰이(60%) + 우측 탐업(40%): 요약/문제/노트/AI 튜터 탐업으로 모든 기능을 회면에 배치합니다.

## 🎯 4. 요구사항 명세서

### 4-1. 기능 요구사항 (Functional Requirements)

#### **FR-01: 회원가입**

| 항목 | 내용 |
|------|------|
| **요구사항** | 이메일과 비밀번호로 계정 생성 |
| **이메일 규칙** | 정식 검증 (user@domain.com). 중복 이메일 가입 불가. |
| **비밀번호 규칙** | 최소 8자 이상, 영문+숫자 포함 필수. 확인란 불일치 시 가입 불가. |
| **저장 방식** | bcrypt (salt rounds=10) 해싱. 평문 저장 금지. |
| **성공 처리** | 계정 생성 후 /login 페이지로 이동. 이메일 인증 없음. |
| **오류 메시지** | 중복 이메일: 이메일 중복 메시지 표시. 비밀번호 규칙 미충족: 비밀번호 불일치 + 비밀번호 규칙 불일치 안내 표시 |

#### **FR-02: 로그인**

| 항목 | 내용 |
|------|------|
| **요구사항** | 이메일·비밀번호로 로그인. 소셜 로그인 제외. |
| **인증 방식** | JWT (Access Token). 만료 시간 24시간. 클라이언트 localStorage에 저장. |
| **성공 처리** | JWT 발급 후 /documents(문서함)로 이동. |
| **실패 처리** | 이메일 미존재 또는 비밀번호 불일치 → '이메일 또는 비밀번호를 확인하세요' 표시 |

#### **FR-03: 로그아웃**

| 항목 | 내용 |
|------|------|
| **요구사항** | 로그아웃 클릭 시 JWT를 localStorage에서 삭제하고 /login으로 이동한다. |
| **서버 처리** | 서버 쪽 토큰 블랙리스트 없음. 클라이언트 토큰 삭제 (MVP 범위). |

#### **FR-04: 회원 탈퇴**

| 항목 | 내용 |
|------|------|
| **요구사항** | 탈퇴 요청 시 해당 계정의 모든 데이터를 영구 삭제한다. |
| **삭제 범위** | users 레코드 삭제 시 ON DELETE CASCADE로 pdfs·summaries·questions·notes 전부 삭제. 서버 /uploads/{userId}/ 디렉토리 파일도 삭제. |
| **확인 절차** | '탈퇴하면 모든 데이터가 삭제됩니다' 확인 다이얼로그 표시 후 진행. |
| **처리 완료** | JWT 삭제 → /login으로 이동. |

#### **FR-05: PDF 업로드**

| 항목 | 내용 |
|------|------|
| **요구사항** | 텍스트 레이어가 있는 PDF를 업로드. 완료 즉시 요약·문제 노트 자동 생성 |
| **파일 제한** | 확장자: .pdf만. 크기: 최대 20MB. 페이지 수: 최대 50페이지. |
| **저장 방식** | 서버 /uploads/{userId}/ 디렉토리에 저장. 파일명은 UUID로 변환 (원본 파일명 DB에 저장). |
| **텍스트 추출** | pdf-parse 라이브러리로 텍스트 추출 → DB pdfs.extracted_text 컬럼에 저장. |
| **자동 실행** | 업로드 후 아래 3가지를 Promise.all로 병렬 실행:
① Claude API 호출 → 요약 생성 → summaries 테이블 저장 (status: processing → done)
② Claude API 호출 → 문제 5개 생성 → questions 테이블 저장 (JSON 문제 포함)
③ 빈 노트 레코드 생성 |
| **오류 메시지** | 파일 크기 초과: .pdf 아닌 파일 / 20MB 초과 / 50페이지 초과 |

#### **FR-06: 문서함 조회**

| 항목 | 내용 |
|------|------|
| **요구사항** | 업로드된 PDF 목록을 최신순으로 조회. 선택 시 학습 화면(/study/:pdfId)으로 이동. |
| **표시 정보** | 원본 파일명, 업로드 날짜, 요약 상태(처리중/완료), 문제 상태(처리중/완료) |
| **정렬 기준** | 업로드 최신순 (created_at DESC). |
| **빈 상태** | 아직 업로드한 문서가 없으면 '문서 추가하기' 유도 메시지 표시. |

#### **FR-07: PDF 삭제**

| 항목 | 내용 |
|------|------|
| **요구사항** | PDF 및 연관 데이터(요약·문제·노트) 모두 삭제한다. |
| **삭제 처리** | pdfs 레코드 삭제 시 ON DELETE CASCADE로 summaries·questions·notes 자동 삭제. 서버 파일(/uploads/{userId}/{uuid}.pdf) 삭제. |
| **확인 절차** | '삭제하면 요약·문제·노트가 모두 사라집니다' 확인 후 진행. |

#### **FR-08: PDF 뷰어**

| 항목 | 내용 |
|------|------|
| **요구사항** | 업로드된 PDF를 화면 좌측(60%)에 렌더링. |
| **라이브러리** | react-pdf (pdfs-dist 기반). npm 설치 필요. |
| **제공 기능** | 좌측/우측 페이지 이동 버튼, 현재 페이지/전체 페이지 수 표시. |
| **제외 기능** | 텍스트 선택, 하이라이트, 북마크 기능 없음. |

#### **FR-09: 탭 패널 전환**

| 항목 | 내용 |
|------|------|
| **요구사항** | 화면 우측(40%)에 4개 탭(요약, 문제, 노트, AI 튜터) 표시. 기본 탭: 요약. |
| **기본 탭** | 화면 진입 시 자동으로 요약 탭 활성화. |
| **레이아웃** | 각 탭 너비 40%, 스크롤 가능. |

#### **FR-10: 요약 탭 조회**

| 항목 | 내용 |
|------|------|
| **요구사항** | 해당 PDF의 요약을 조회. 요약이 아직 생성 중이면 '요약을 생성하고 있습니다' 스피너 표시. 생성 완료 후 텍스트 표시. |
| **상태별 처리** | status = 'processing' → 스피너 + 3초마다 폴링. status = 'done' → 한국어 요약 텍스트 표시 (14px, line-height 1.8). status = 'failed' → ti-alert-circle 아이콘 + '요약 생성에 실패했습니다' + [재시도] 버튼 |
| **언어 처리** | 영어 PDF는 변역+요약을 Claude에게 요청 → 한국어 결과만 표시. 변역 결과는 별도 표시 없음. |
| **한국어 요약 전환** | System: '당신은 학습 도우미입니다.' User: '다음 내용을 한국어로 요약하세요: [extracted_text]' |

#### **FR-11: 문제 탭 조회**

| 항목 | 내용 |
|------|------|
| **요구사항** | 해당 PDF의 문제를 조회. 문제가 아직 생성 중이면 스피너 표시. 생성 완료 후 4선지 객관식 5개 문제를 카드 형식으로 표시. |
| **Claude 프롬프트** | System: '당신은 확실 도우미입니다.' User: '다음 내용을 바탕으로 확실 확인을 4지선다 객관식 문제 5개 한국 어로 만들어주세요: [extracted_text]' |
| **응답 형식** | JSON 문제 집합. [{"question":"", "options":["①","②","③","④"], "answer":0}] × 5개. answer는 정답 보기 인덱스(0~3). |
| **클라이언트 처리** | answer 필드는 DB에 저장하지만, API 응답 시 제거. 사용자가 응답 선택 시 검증. |
| **표시 형식** | 정답(answer) 미표시. 문제 번호 + 질문 텍스트 (14px bold). ① ② ③ ④ 보기 4개 (14px). 카드 간 구분선. |

#### **FR-12: 노트 탭 작성/수정**

| 항목 | 내용 |
|------|------|
| **요구사항** | PDF에 종속된 노트를 조회·수정. 입력 후 2초 후 자동 저장 (debounce 2000ms). |
| **자동 저장** | 입력 후 2초 주기 자동 저장. debounce 처리로 불필요한 요청 제거. |
| **초기 상태** | PDF 입로드 시 빈 노트 레코드 생성. placeholder: '여기에 노트를 작성하세요...' |
| **저장 로직** | GET /api/pdfs/:pdfId/note → content 표시, 빈 문자열이면 placeholder 표시. 입력 2초 후 → PUT /api/pdfs/:pdfId/note 자동 호출. |

### 도메인 개요

본 시스템은 4가지 핵심 도메인으로 구성됩니다:

| 도메인 영역 | 설명 | 포함 개념 |
|-----------|------|---------|
| **회원 관리** | 사용자 계정 생성, 로그인, 탈퇴 | User, 인증/인가 |
| **PDF 관리** | PDF 업로드, 조회, 삭제 | PDF, 파일 저장소 |
| **학습 콘텐츠** | 자동 생성 요약, 문제, 노트 | Summary, Questions, Note |
| **AI 연동** | Claude API 기반 기능 | AI 튜터, 요약 생성 |

### 도메인 개념 사전

| 용어 | 정의 | 비고 |
|------|------|------|
| **User (사용자)** | 이메일·비밀번호로 가입한 서비스 이용자 | 비로그인 사용자는 User가 아님 |
| **PDF** | 사용자가 업로드한 강의 자료 PDF 파일 | .pdf 확장자, 20MB·50페이지 제한 |
| **extracted_text** | pdf-parse로 추출한 PDF 전체 텍스트 | DB에 저장, AI 튜터 기반 자료 |
| **Summary (요약)** | Claude API가 생성한 한국어 요약 텍스트 | status: processing / done / failed |
| **Question (문제)** | Claude API가 생성한 4선지 객관식 문제 | JSON 문제 집합. 정답은 숨김 |
| **Note (노트)** | 사용자가 직접 작성한 학습 노트 | debounce 2000ms로 자동 저장 |
| **JWT** | 로그인 후 발급하는 인증 토큰 | 24시간 만료 |
| **status** | 요약/문제의 생성 상태 | processing / done / failed |
| **SSE** | Server-Sent Events | 단방향 스트리밍 |

| **기본 흐름** | 1. 시스템이 GET /api/pdfs/:pdfId/summary 호출. 2. status = 'processing'이면 스피너 표시 + 3초마다 폴링. 3. status = 'done'이면 한국어 요약 텍스트 표시 (14px, line-height 1.8). 4. status = 'failed'이면 ti-alert-circle 아이콘 + '요약 생성에 실패했습니다' + [재시도] 버튼. |
| **대안-에이** 흐름 | 4a. [재시도] 클릭 → POST /api/pdfs/:pdfId/summary/retry → status='processing' 재설정 → 3초마다 폴링. |
| **사후 조건** | 요약 텍스트 또는 오류 메시지 표시. |

| **대안-에이** 흐름 | 3a. Claude API 오류 → '응금을 가져오지 못했습니다' + [재시도] 버튼 |

## 📈 9. API 엔드포인트 설계

### 공통 규칙

- **Base URL**: http://localhost:4000/api
- **인증**: Authorization: Bearer {JWT}
- **Content-Type**: application/json (또는 multipart/form-data)
- **오류 응답**: { "error": "오류 메시지" }
- **성공 응답**: { "data": { ... } }
